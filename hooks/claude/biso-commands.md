# biso command catalog

You are running inside (or alongside) a biso PTY. The following commands
are available IF `$BISA_URL` and `$BISA_TOKEN` are set in env. Verify
with `biso status` before use. If `biso` is not on PATH, ignore this
file — the env vars alone don't enable the catalog.

This file is managed by `biso install-hook` — do not edit; changes will
be overwritten on the next install. To remove, run
`biso install-hook --remove --purge`.

## logging — default is under-log

Tag with the appropriate type. Always pair with `proj/<id>`. Quick
classification:

- `decision`  — A chosen over B, with reason
- `learning`  — non-obvious finding / root cause / API quirk
- `blocker`   — hit friction >15 min and how you unstuck
- `milestone` — feature shipped / PR merged / release cut
- `bug`       — root cause (not symptom)
- `userpref`  — user corrected you or stated a preference
- `redirect`  — scope changed mid-task

Examples:

    biso log "chose X over Y — reason" --tag decision --tag proj/<id>
    biso log "non-obvious finding"     --tag learning  --tag proj/<id>
    biso log --silent --tag milestone  "feature shipped"

**Never log:** narration of intent, routine tool-call successes, code
snippets, same-turn errors, progress updates. Most turns produce zero
entries. If unsure, skip.

## retrieval — search before re-deriving

Before making a non-trivial decision on something that might have prior
context (architecture choice, debugging path, recurring pattern), ask
the journal:

    biso ask "what did I decide about terraform module structure?"
    biso ask "milestones per project" --days 30
    biso ask "onboarding status" --project <id> --days 14

Skip for one-shot routine operations. Don't call on every prompt — noise
drowns signal.

## cross-machine — alternatives to scp / rsync / ssh

Replace ad-hoc scp/rsync/ssh into other machines you control:

    biso put runbook.md --tag capture       # upload to codex/refs/
    biso get scripts/eks-bootstrap          # pull a file/dir
    biso run scripts/deploy.sh --env stg    # remote exec (uses project execTarget)
    biso run --bundle ./pipeline --entry main.sh

If the active project has `execTarget` set in `.meta/projects.json`,
`biso run` is the contract — never `bash <script>` / `ssh other 'bash...'`.

## inspection

    biso status                            # server health + current project
    biso journal today
    biso projects
    biso loop status
    biso gain                              # token + cost analytics
    biso discover                          # adoption gap report
    biso learn                             # CLI correction patterns

## echoes — surface relevant prior journal entries

Closes the journal→agent feedback loop. Before non-trivial decisions:

    curl -sS -H "x-bisa-token: $BISA_TOKEN" \
      "$BISA_URL/codex/echoes?q=<your-question>&caller=<your-short-id>"

Always pass `caller=<short-id>` so efficacy telemetry attributes calls.
